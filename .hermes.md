# drivexam Project Rules

## Project Defaults

- Project path: `D:\Factory\drivexam`.
- Inspect the current stack before assuming framework, database, auth, deployment, billing, analytics, or AI-provider choices.
- Keep changes small, phased, and verified.
- Do not push, deploy, send emails, or make external side-effect changes unless Louie explicitly approves.

## Verification

Before calling technical work complete, run the relevant checks once the stack exists:

- Build command, if configured.
- Tests, if configured.
- Lint/typecheck, if configured.
- Browser verification for visual changes.
- Migration checks for database changes.

If a command cannot be run, explain why and what should be checked manually.

## Handoff

Use concise shipped-change summaries with:

1. What changed
2. Files changed
3. Commands run / verification result
4. Risks or notes
5. Deployment or migration handoff, if relevant
6. Recommended next step
